# Prompt：main.go 结构拆分分析与方案生成

你是资深 Go 架构工程师。请阅读当前项目的 `main.go`，并给出系统性的拆分方案与迁移步骤，目标是减少单文件体积、提升可维护性和可测试性，同时保持现有功能不变。

## 输出要求
1. 给出清晰的模块划分与文件结构树（例如 `cmd/`, `internal/`, `pkg/` 等）。
2. 明确每个新文件/包的职责边界与公开接口。
3. 列出需要迁移的函数/结构体清单（按模块分组）。
4. 说明跨模块依赖与数据流向（例如配置 -> 扫描 -> 去重 -> 计划 -> 报告）。
5. 给出逐步迁移计划（分阶段、可回滚）。
6. 指出潜在的循环依赖与避免策略。
7. 拆分后需保证 CLI 参数与行为不变。

## 当前功能要点（必须覆盖）
- CLI 参数解析与校验
- 文件扫描（视频/非视频）
- 哈希计算、近重复聚类
- 抽帧与 AI 识别（Ollama）
- 计划输出（JSON/CSV）与执行移动
- 报告输出与日志
- 进度条
- Windows 控制台编码修复

## 约束
- 不引入新的第三方依赖
- 兼容当前运行命令与参数
- 拆分后需易于单元测试（尤其是去重逻辑、计划生成、AI 调用）
