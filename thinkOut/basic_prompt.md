# 视频去重与省市归类项目基础 Prompt

## 角色
你是资深 Golang 工程师与计算机视觉工程师，负责实现一个可在本地运行的视频去重与归类工具。

## 项目目标
- 递归读取指定文件夹及其所有子文件夹中的视频文件
- 对视频进行相似度比对，识别重复或近重复视频并去重
- 基于视频所在文件夹路径提取省份与城市信息，并按“省份/城市”目录结构整理输出
- 省市信息来源于目录结构，不通过视频内容识别
- 若省份或城市无法确定，则按输入中的原有目录结构放置
- 非视频文件不参与分析，但要保持原目录结构同步归档

## 已知环境
- 操作系统：Windows
- 开发语言：Golang
- Ollama 地址：`http://localhost:11434/`
- 模型：`qwen3-vl:8b`

## 输入与输出
### 输入
- `inputDir`：待处理视频根目录
- `outputDir`：整理后的输出目录
- 可选参数：并发数、采样帧间隔、相似阈值、最小时长过滤、是否保留重复副本

### 输出
- 按 `outputDir/省份/城市/` 归档后的视频文件
- 若省份或城市未知，则按输入中的原有目录结构放置在输出文件夹中
- 非视频文件按输入中的原有目录结构放置在输出文件夹中（不参与分析）
- 重复视频清单（包含原始路径、保留文件、相似度）
- 处理日志（成功、失败、跳过原因）

## 技术要求
- 使用 Golang 实现，结构清晰，模块可扩展
- 视频遍历支持常见格式：`.mp4 .mov .mkv .avi .flv .wmv`
- 先基于哈希摘要算法（如 SHA-256/SHA-512）进行第一轮去重，再进行精细相似度判定
- 提供可配置阈值，避免误删
- 去重策略默认“保留质量更高或时长更完整版本”
- 所有关键步骤要有错误处理与重试机制
- 未知省市时不放入“待分类/未知城市”，改为按原目录回落放置
- 非视频文件跳过去重与识别流程，仅执行目录保持与搬运

## 推荐实现思路
1. 扫描阶段：递归收集视频文件与基础元数据
2. 摘要阶段：计算文件级哈希摘要（如 SHA-256）用于第一轮精确去重
3. 抽帧阶段：按固定间隔提取关键帧，生成帧样本
4. 判重阶段：对哈希未命中但疑似重复的候选做精细比对，输出相似度与保留决策
5. 路径解析阶段：从文件夹层级解析省份/城市信息并做标准化，未知则回落原目录
6. 归类阶段：视频按省份/城市或原目录回落规则归档，非视频文件按原目录直接归档
7. 报告阶段：导出去重清单、归类结果、失败任务

## 调用模型约束
- 仅使用本地 Ollama，不依赖云端 API
- 省份与城市归类不依赖模型识别，仅基于文件夹路径
- 仅在需要时将模型用于辅助任务，不作为省市判断依据
- 对省市名称做标准化映射，避免同名异写
