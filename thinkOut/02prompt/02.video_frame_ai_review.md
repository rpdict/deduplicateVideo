# 需求分析：720p单帧AI识别 + 复核文档拷贝 + 环境识别归类

## 背景与目标
使用本地模型 `qwen3-vl:8b` 对视频进行图像识别。每个视频需先转为 720p 单帧图像，再交由 AI 识别。识别结果必须生成一个**可读、可由程序解析**的文档，用户审核确认后，再执行文件拷贝/归档动作。

地点（省/市）以**文件夹名称**为主，AI 识别结果用于补充“环境类型”（如雪山、沙漠、海边、城市夜景等），用于目录摆放的**二级分类**或**附加标签**。

## 核心流程
1. 扫描输入目录，识别视频文件列表。
2. 对每个视频抽取**单帧**并统一转换为 **720p**。
3. 将该帧提交给 `qwen3-vl:8b` 进行图像识别，输出结构化结果。
4. 生成“待执行文档”（可读、可由程序解析）。
5. 用户人工审核该文档并确认。
6. 程序根据文档执行文件拷贝/归档。

## 识别与归类规则
### 地点归类
- **主要来源**：输入目录中的省/市文件夹名称。
- 若目录中可解析出省/市，则优先使用目录信息。
- 若目录不可解析（或为空），按**原有目录结构**回落放置。

### 环境识别
- AI 识别输出环境类型（如：雪山、草原、海滩、沙漠、城市、森林、湖泊等）。
- 环境类型用于**二级分类**或**标签**：
  - 示例：`输出/浙江省/杭州市/雪山/xxx.mp4`
  - 若环境识别失败，则按输入中的原有目录结构放置在输出文件夹中。

## 视频帧提取要求
- 每个视频提取 **1 帧**。
- 分辨率统一为 **720p**（1280x720）。
- 建议使用 `ffmpeg` 或同类工具：
  - 示例：`ffmpeg -i input.mp4 -vf "scale=1280:720" -frames:v 1 output.jpg`

## AI 识别输出格式（建议）
**输出需结构化，方便程序读取：**

```json
{
  "source_path": "D:/input/浙江省/杭州市/xxx.mp4",
  "frame_path": "D:/tmp/frames/xxx.jpg",
  "province": "浙江省",
  "city": "杭州市",
  "environment": "雪山",
  "confidence": 0.86
}
```

## 待执行文档格式（建议）
用于用户审核与程序执行复制：

```json
{
  "source_path": "D:/input/浙江省/杭州市/xxx.mp4",
  "dest_path": "D:/output/浙江省/杭州市/雪山/xxx.mp4",
  "environment": "雪山",
  "province": "浙江省",
  "city": "杭州市",
  "approved": false
}
```

用户审核后在命令行输入 yes/no，确认后程序才执行文件拷贝/归档。

## 关键约束
- **AI 只做环境识别**，省/市仍以目录为准。
- 未识别省市 → 原目录结构回落。
- 非视频文件不参与识别，但按原目录结构同步归档。
- “待执行文档”必须可机器解析，推荐 JSON/CSV。

## 输出产物
1. 识别结果文档（识别阶段）
2. 待执行文档（用户审核阶段）
3. 执行日志与失败清单

## 风险与对策
- **误判环境**：提供 `confidence` 字段，低于阈值则按原目录结构回落放置。
- **用户漏审**：默认 `approved=false`，避免误拷贝。
- **帧提取失败**：记录错误并跳过该视频。
